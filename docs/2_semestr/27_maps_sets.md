# Лекція 27: Асоціативні контейнери — Map та Set

[← Лекція 26](26_tree_structures.md) | [Index](index.md)

## Мета

Зрозуміти, як влаштовані **асоціативні контейнери** в C++ STL: `std::map`, `std::unordered_map`, `std::set`, `std::unordered_set`. Навчитися обирати між ними в залежності від задачі. Зрозуміти зв'язок з попередньо вивченими Red-Black Trees та Hash Tables.

## Експрес-опитування

1. Як знайти значення за ключем-рядком (наприклад, номер телефону за ім'ям) у `std::vector`? Яка складність пошуку?
2. Як перевірити унікальність елемента (наприклад, чи користувач із таким ID вже зареєстрований) у `std::vector`? Яка складність?

<details markdown="1">
<summary>Інженерна відповідь</summary>

1. Вектор індексується лише цілими числами. Доведеться зберігати пари на кшталт `std::pair<string, string>` і робити лінійний пошук за $O(N)$. Це приводить до необхідності **Map** (словника).
2. Доведеться щоразу робити лінійний пошук по всьому вектору за $O(N)$, щоб переконатися, що такого ID ще немає. Для $N$ реєстрацій це дасть загальну складність $O(N^2)$. Це приводить до необхідності **Set** (множини).

</details>

## Частина 1: std::set — Множина унікальних елементів

### Що таке Set?

**Множина (Set)** — контейнер, що зберігає **унікальні** елементи у **відсортованому** порядку. Математична аналогія з теорії множин.

```
Ключові властивості std::set:
  ✓ Унікальність:   дублікати ігноруються
  ✓ Відсортованість: елементи завжди у порядку (asc)
  ✓ Реалізація:     Red-Black Tree (L26!)
```

**Складність операцій:**

| Операція | Час | Пояснення |
|---------|-----|-----------|
| `insert(x)` | O(log n) | Вставка елемента в дерево (пошук місця + вставка) |
| `erase(x)` | O(log n) | Видалення елемента (пошук вузла + видалення + ребалансування) |
| `find(x)` | O(log n) | Бінарний пошук у дереві |
| `count(x)` | O(log n) | Кількість входжень (пошук у дереві, повертає 0 або 1 через унікальність) |
| `begin()/end()` | O(1) | Отримання ітераторів на початок/кінець (для обходу) |
| `size()` | O(1) | Кількість елементів (зберігається окремо) |

### Синтаксис та основні операції

```cpp
#include <set>

std::set<int> s;

// Вставка
s.insert(5);
s.insert(2);
s.insert(8);
s.insert(2);  // Ігнорується — дублікат!
s.insert(1);

// Стан: {1, 2, 5, 8} — відсортовано автоматично!

// Пошук (find повертає ітератор)
auto it = s.find(5); // auto розгортається в тип std::set<int>::iterator
if (it != s.end()) {
    std::cout << "Знайдено: " << *it; // 5
}

// count() — зручна перевірка наявності
if (s.count(3) == 0) {
    std::cout << "3 відсутній";
}

// Видалення
s.erase(2);  // {1, 5, 8}

// Обхід (завжди в порядку!)
for (int x : s) {
    std::cout << x << " "; // 1 5 8
}

// Розмір
std::cout << s.size(); // 3
```

### Range queries (діапазонний пошук)

`std::set` завдяки сортуванню підтримує ефективний пошук у діапазоні:

```cpp
std::set<int> s = {1, 3, 5, 7, 9, 11, 13};

// lower_bound(x) — перший елемент >= x (початок діапазону)
auto lo = s.lower_bound(5);  // вказує на 5

// upper_bound(x) — перший елемент > x (елемент одразу за кінцем діапазону)
// Щоб включити 9 у вибірку, верхня межа має вказувати на перший елемент, що є строго більшим за 9.
auto hi = s.upper_bound(9);  // вказує на 11

// Напіввідкритий інтервал [lo, hi) містить {5, 7, 9} (11 не входить у цикл)
for (auto it = lo; it != hi; ++it) {
    std::cout << *it << " "; // Виведе: 5 7 9
}
```

### Коли використовувати std::set?

- ✅ Потрібна **автоматична сортованість** елементів.
- ✅ Потрібні **range-based queries** (діапазонний пошук від A до B за допомогою `lower_bound`/`upper_bound`).
- ⚠️ **Дедуплікація** (якщо елементів небагато і сортування все одно потрібне. Для великих обсягів даних значно ефективніше застосувати комбінацію `std::sort` + `std::unique` на векторі, оскільки `std::set` створює окремий вузол у купі на кожен елемент, що має величезний оверхед по пам'яті та кеш-промахах).
- ❌ Не підходить, якщо потрібен швидкий доступ за індексом за $O(1)$.

## Частина 2: std::unordered_set — Хеш-таблиця

### Що всередині?

`std::unordered_set` побудований на **хеш-таблиці** замість дерева. Відсутній порядок, зате O(1) у середньому.

> [!NOTE]
> **🧠 Що таке хешування? (Коротка довідка)**
> 
> Оскільки ви ще не проходили алгоритми хешування, ось основні концепції простими словами:
> 
> * **Хеш-функція** — це функція, яка приймає будь-які дані (рядок, число, гігантський файл) і перетворює їх на одне число фіксованого розміру (**хеш-код**). Це як «цифровий відбиток» даних.
> * **Головне правило:** Для однакових даних хеш завжди однаковий. Якщо змінити хоча б один символ — хеш зміниться до невпізнаваності.
> * **Колізія** — ситуація, коли два *різні* об'єкти отримують *однаковий* хеш (наприклад, `hash("apple") == hash("banana")`). Оскільки варіантів вхідних даних нескінченно багато, а кількість можливих хешів обмежена, колізії неминучі.
> 
> **Де ви зустрічаєте хешування в інженерії:**
> * **MD5 / CRC32** — швидкі хеші для перевірки цілісності файлів. Коли ви завантажуєте файл з інтернету, ви можете звірити його MD5-хеш із сайтом автора, щоб переконатися, що файл не пошкоджений.
> * **SHA-256 / SHA-3** — криптографічні хеші. Вони спроектовані так, щоб колізію було математично неможливо знайти штучно. Використовуються для безпечного збереження паролів у БД (зберігають лише хеш, а не сам пароль) та у блокчейні (Bitcoin).
> * **`std::hash`** — вбудована в C++ швидка некриптографічна функція. Використовується під капотом `std::unordered_set` / `std::unordered_map` для швидкого розкидання елементів по комірках.
> * **⚠️ Важливе обмеження (довжина ключа):** Час обчислення хеш-функції прямо пропорційний розміру ключа. Якщо ключем є рядок довжиною $L$, то обчислення його хешу займе $O(L)$ операцій. Тому пошук у хеш-таблиці працює за $O(1)$ лише за умови, що розмір ключа невеликий (константний). Для дуже довгих рядків (наприклад, цілих абзаців тексту) хешування стає відчутно повільнішим.

```
Порівняння std::set vs std::unordered_set:

std::set (RB-Tree):
  {1, 2, 5, 8} → завжди відсортовано
  insert/find: O(log n)

std::unordered_set (Hash Table):
  {8, 1, 5, 2} → порядок довільний (залежить від хешу)
  insert/find: O(1) average*, O(n) worst case
  * Примітка: O(1) тут означає незалежність від кількості елементів (n) у контейнері. Проте реальний час пошуку залежить від довжини самого ключа: для рядка довжиною L пошук вимагатиме O(L) операцій (для обчислення хешу та порівняння значень).
```

```cpp
#include <unordered_set>

std::unordered_set<std::string> us;

us.insert("apple");
us.insert("banana");
us.insert("cherry");
us.insert("apple");   // дублікат, ігнорується

std::cout << us.count("banana"); // 1 (є)
std::cout << us.count("mango");  // 0 (немає)

us.erase("banana");

// ВАЖЛИВО: порядок обходу НЕ гарантований!
for (const auto& s : us) {
    std::cout << s << " "; // може бути будь-який порядок
}
```

### Коли що обирати?

| Критерій | `std::set` | `std::unordered_set` |
|---------|-----------|---------------------|
| **Порядок** | Відсортований | Немає |
| **Пошук** | O(log n) | O(1) avg |
| **Range queries** | ✅ Підтримує | ❌ Ні |
| **Worst-case** | O(log n) гарантовано | O(n) при колізіях (наприклад, якщо написати власну погану хеш-функцію, яка повертає однакові значення для різних ключів) |
| **Пам'ять** | Менше (виділяє пам'ять точно під кожен елемент) | Більше (через резервування порожніх комірок — *load factor*) |
| **Коли** | Потрібен порядок або діапазони | Тільки пошук/вставка |

*\* Примітка про пам'ять та **load factor** (коефіцієнт завантаження):* Хеш-таблиця працює швидко ($O(1)$) лише тоді, коли в ній багато вільних комірок. Якщо кількість елементів наближається до кількості комірок (коефіцієнт завантаження стає великим, наприклад > 0.75), відбуваються колізії. Тому `std::unordered_set` завжди тримає значну частину пам'яті порожньою (резервує комірки із запасом).

## Частина 3: std::map — Словник (ключ → значення)

### Що таке Map?

**Map (Асоціативний масив / Словник / Словниковий ADT)** — це абстрактний тип даних (**ADT**), який зберігає унікальні пари **ключ → значення**.

* **Математична суть ADT:** Це відображення (mapping) множини унікальних ключів на множину значень. Сам по собі ADT не накладає жодного порядку на ці пари.
* **Реалізація C++ (`std::map`):** На відміну від чистого ADT, конкретна реалізація `std::map` у C++ завжди зберігає пари відсортованими за їхнім ключем (завдяки використанню Червоно-Чорного дерева під капотом).

```
std::map — це std::set, але кожен елемент має "прикріплене" значення.

Колекція: { "Alice" → 95, "Bob" → 87, "Charlie" → 92 }
Ключ: std::string (ім'я)
Значення: int (оцінка)
```

> [!TIP]
> **💡 Цікавий факт (Звідки взялися Alice, Bob та Charlie?):**
> 
> У криптографії, програмуванні та теорії мереж імена **Alice**, **Bob** та **Charlie** (а також **Eve** — шпигун, від *eavesdropper*) є стандартними персонажами для опису протоколів. Цю традицію розпочали автори алгоритму шифрування RSA у 1978 році, щоб замінити сухі та нудні терміни «Суб'єкт А» та «Суб'єкт Б» (які використовувалися в радіозв'язку та телекомунікаціях) на живі імена, які легко запам'ятовуються. Вони просто відповідають першим літерам латинського алфавіту: A, B, C...

**Складність:** Ідентична `std::set` — O(log n) для пошуку/вставки/видалення.

### Синтаксис та ключові операції

```cpp
#include <map>

std::map<std::string, int> scores;

// Вставка — 3 способи:
scores["Alice"] = 95;                         // 1. operator[] (вставляє якщо немає!)
scores.insert({"Bob", 87});                   // 2. insert з pair
scores.emplace("Charlie", 92);                // 3. emplace (конструює елемент прямо в пам'яті дерева, без створення тимчасової пари)

// УВАГА: operator[] - небезпечний при читанні!
// Якщо ключ відсутній — він АВТОМАТИЧНО ДОДАЄТЬСЯ з value = 0
auto val = scores["Dave"];  // Тепер Dave є в map з оцінкою 0!
std::cout << scores.size(); // 4, не 3!

// Безпечний пошук:
auto it = scores.find("Alice"); // auto розгортається в тип std::map<std::string, int>::iterator
if (it != scores.end()) {
    std::cout << it->first << ": " << it->second; // Alice: 95
}

// Перевірка наявності (без вставки!):
if (scores.count("Bob") > 0) {
    std::cout << "Bob exists";
}

// Видалення
scores.erase("Dave"); // видаляємо випадково доданий

// Обхід (ключі завжди відсортовані!)
for (const auto& [name, score] : scores) {  // C++17 structured binding
    std::cout << name << " → " << score << "\n";
    // Alice → 95
    // Bob → 87
    // Charlie → 92
}
```

> [!WARNING]
> **Класична пастка:** `map["key"]` при **читанні** невідомого ключа **вставляє** його з нульовим значенням! Використовуйте `find()` або `count()` для безпечної перевірки.

> [!NOTE]
> **Різниця між `insert` та `emplace`:**
> * `insert({"Bob", 87})` вимагає створення тимчасового об'єкта пари (`std::pair`), який потім копіюється або переміщується у новостворений вузол дерева.
> * `emplace("Charlie", 92)` створює об'єкт пари безпосередньо у виділеній під вузол дерева пам'яті, уникаючи зайвих операцій копіювання/переміщення тимчасових об'єктів.

### Практичний приклад: Підрахунок частоти слів

```cpp
#include <map>
#include <string>
#include <sstream>

std::map<std::string, int> wordCount(const std::string& text) {
    std::map<std::string, int> freq;
    std::istringstream iss(text);
    std::string word;
    
    while (iss >> word) {
        freq[word]++;  // якщо слова нема — створює з 0, потім +1
    }
    return freq;
}

auto freq = wordCount("the cat sat on the mat the cat");
// { "cat":2, "mat":1, "on":1, "sat":1, "the":3 } — відсортовано!

for (auto& [word, count] : freq) {
    std::cout << word << ": " << count << "\n";
}
```

## Частина 4: std::unordered_map — Невпорядкований словник (хеш-таблиця)

### Зв'язок з ADT та структурою даних

* **ADT (абстрактний тип):** `std::unordered_map` реалізує той самий **Словниковий ADT (Map)**, що й `std::map`. З точки зору інтерфейсу користувача вони майже ідентичні (обидва зберігають пари унікальний ключ → значення).
* **Структура даних (реалізація):** Під капотом замість збалансованого дерева використовується **Хеш-таблиця**. Це структура даних, яка безпосередньо відображає ключі на індекси в масиві пам'яті за допомогою хеш-функції, забезпечуючи доступ за константний середній час $O(1)$.
  
  **Як конкретно працює пошук за ключем у хеш-таблиці:**
  1. Отримує ключ (наприклад, рядок `"Alice"`).
  2. Обчислює для нього числовий хеш-код: `h = hash("Alice")` (наприклад, `192837465`).
  3. Знаходить індекс комірки (bucket) у внутрішньому масиві через остачу від ділення на розмір масиву: `index = h % bucket_count`.
  4. Переходить безпосередньо за адресою `array[index]` для запису чи читання (тому це працює за константний час).

### Різниця з std::map

`std::unordered_map` — це інша реалізація того самого словникового ADT, яка використовує хеш-таблицю замість Червоно-Чорного дерева.

```cpp
#include <unordered_map>

std::unordered_map<std::string, int> phoneBook;

phoneBook["Alice"] = 380501234567;
phoneBook["Bob"]   = 380671234567;

// Пошук — O(1) середньому!
if (phoneBook.count("Alice")) {
    std::cout << phoneBook["Alice"]; // 380501234567
}

// Обхід — порядок НЕ гарантований:
for (auto& [name, phone] : phoneBook) {
    std::cout << name << ": " << phone << "\n";
    // Може бути Bob перед Alice!
}
```

### Порівняльна таблиця всіх 4 контейнерів

| Контейнер | Унікальність | Порядок | Пошук | Структура |
|-----------|-------------|---------|-------|-----------|
| `std::set` | ✅ ключів | ✅ відсортований | O(log n) | RB-Tree |
| `std::unordered_set` | ✅ ключів | ❌ | O(1) avg | Hash Table |
| `std::map` | ✅ ключів | ✅ відсортований за ключем | O(log n) | RB-Tree |
| `std::unordered_map` | ✅ ключів | ❌ | O(1) avg | Hash Table |

## Частина 5: Вибір контейнера — Дерево рішень

```
Потрібне зберігання пар (ключ, значення)?
│
├── НІ  → Потрібна унікальність?
│         ├── НІ  → std::vector або std::list
│         └── ТАК → Потрібен відсортований порядок?
│                   ├── ТАК → std::set
│                   └── НІ  → std::unordered_set  (швидший)
│
└── ТАК → Потрібен відсортований порядок або range queries?
          ├── ТАК → std::map
          └── НІ  → std::unordered_map  (швидший)
```

## Контрольні питання

1. Чому `scores["Dave"]` у `std::map` додає новий запис, а `scores.find("Dave")` — ні?

<details markdown="1">
<summary>Відповідь</summary>

`operator[]` — це "доступ або вставка". Якщо ключ відсутній, він конструює новий елемент з default value (0 для int). `find()` лише шукає і повертає `end()` якщо не знайдено, без жодної вставки.

</details>

2. Навіщо використовувати `std::set` замість `std::unordered_set`, якщо у другого краща складність?

<details markdown="1">
<summary>Відповідь</summary>

`std::set` підходить коли: потрібен **відсортований порядок** (наприклад, вивести в алфавітному порядку), потрібні **range queries** (`lower_bound`, `upper_bound`), або потрібна **гарантована** O(log n) (без worst-case O(n) при хеш-колізіях).

</details>

3. Як ефективно знайти 5 найпопулярніших слів у тексті?

<details markdown="1">
<summary>Відповідь</summary>

1. Підрахуємо частоту через `std::unordered_map<string, int>` — O(n)
2. Завантажимо пари (count, word) в `std::priority_queue` розміром 5 — O(n log 5) = O(n)
3. Витягнемо top-5 з черги — O(5 log 5) = O(1)

Загальна складність: O(n), набагато краще ніж сортування O(n log n).

</details>
